# 官网发布质量修复实施计划

> **供自动化执行者使用：** 必须按任务顺序执行，每项完成后独立验证，再处理下一项。

**目标：** 修复官网在无脚本和移动端的可用性问题，更新存在漏洞的构建依赖，并完善内容一致性、性能、搜索展示与隐私声明。

**架构：** 项目保持 Astro 静态站点和组件化结构。按相互独立的风险项逐项修复：先确保核心内容可访问，再改善导航与依赖安全，最后处理内容、资源和元信息。

**技术栈：** Astro、静态 HTML/CSS、少量原生浏览器脚本、npm。

## 全局约束

- 所有页面必须能在静态输出中直接呈现核心内容。
- 不引入后端、CMS 或不必要的客户端框架。
- 每项用户可见改动都要执行构建与浏览器验证。
- 文案以已发布游戏的真实状态为准，未确认信息不虚构。
